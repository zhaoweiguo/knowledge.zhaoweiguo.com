2.3.1. etcdе‘Ҫд»ӨВ¶

жңү3з§Қй…ҚзҪ®ж–№ејҸ:

1. configuration file
2. various command-line flags
3. environment variables

зҺҜеўғеҸҳйҮҸж јејҸ:

ETCD_еҗҺйқўи·ҹеҜ№еә”еӨ§еҶҷзҡ„flagпјҢеҰӮ:
--my-flag   => ETCD_MY_FLAG
--config-file flag => ETCD_CONFIG_FILE
...

дҪҝз”Ёй…ҚзҪ®ж–Үд»¶жіЁж„Ҹ:

1. й…ҚзҪ®ж–Үд»¶дҪҝз”Ё: --config-file flag
2. дҪҝз”Ёй…ҚзҪ®ж–Үд»¶ж—¶е…¶д»–flagдјҡиў«еҝҪз•ҘпјҢеҰӮ:
   $ etcd --config-file etcd.conf.yml.sample --data-dir /tmp
   // will ignore the --data-dir flag

Member flagsВ¶

вҖ“name:

default: вҖңdefaultвҖқ

вҖ“data-dir:

ж•°жҚ®дҝқеӯҳзҡ„и·Ҝеҫ„

вҖ“heartbeat-interval:

default: вҖң100вҖқ(milliseconds)

вҖ“election-timeout:

default: вҖң1000вҖқ(milliseconds)
йҮҚж–°жҠ•зҘЁзҡ„и¶…ж—¶ж—¶й—ҙпјҢеҰӮжһң follower еңЁиҜҘж—¶й—ҙй—ҙйҡ”жІЎжңүж”¶еҲ°еҝғи·іеҢ…пјҢдјҡи§ҰеҸ‘йҮҚж–°жҠ•зҘЁ
жіЁ: жіЁж„ҸжӯӨеҖјдёҺ--heartbeat-intervalеҖјзҡ„е…ізі»

вҖ“listen-peer-urls:

default: вҖңhttp://localhost:2380вҖқ

е’ҢеҗҢдјҙйҖҡдҝЎзҡ„ең°еқҖ
List of URLs to listen on for peer traffic

вҖ“listen-client-urls:

List of URLs to listen on for client traffic.

Clustering flagsВ¶

вҖ“initial-advertise-peer-urls:

default: вҖңhttp://localhost:2380вҖқ

иҜҙжҳҺ:
  иҜҘиҠӮзӮ№еҗҢдјҙзӣ‘еҗ¬ең°еқҖпјҢиҝҷдёӘеҖјдјҡе‘ҠиҜүйӣҶзҫӨдёӯе…¶д»–иҠӮзӮ№
  List of this memberвҖҷs peer URLs to advertise to the rest of the cluster.
  These addresses are used for communicating etcd data around the cluster.
  At least one must be routable to all cluster members.

вҖ“initial-cluster:

default: вҖңdefault=http://localhost:2380вҖқ
ж јејҸ: <name1>=<peer-urls1>,<name2>=<peer-urls2>,...
жіЁ:
  <name1>: жҳҜиҠӮзӮ№з”Ё--nameжҢҮе®ҡзҡ„еҗҚеӯ—
  <peer-urls1>: жҳҜз”Ё--initial-advertise-peer-urls жҢҮе®ҡзҡ„еҖј

иҜҙжҳҺ:
  Initial cluster configuration for bootstrapping.
  еңЁеҗҜеҠЁж—¶еҲқдҪҝеҢ–йӣҶзҫӨй…ҚзҪ®(еҚі: йӣҶзҫӨдёӯжүҖжңүиҠӮзӮ№зҡ„дҝЎжҒҜ)

вҖ“initial-cluster-state:

default: вҖңnewвҖқ
еҸҜз”ЁеҖј: (вҖңnewвҖқ or вҖңexistingвҖқ)

иҜҙжҳҺ:
    Initial cluster state
    1. new: ж–°е»әйӣҶзҫӨж—¶:
    Set to new for all members present during initial static or DNS bootstrapping.
    2. existing: е·ІеӯҳеңЁйӣҶзҫӨж—¶:
    If this option is set to existing, etcd will attempt to join the existing cluster.

вҖ“initial-cluster-token:

default: вҖңetcd-clusterвҖқ

иҜҙжҳҺ:
  еҲӣе»әйӣҶзҫӨзҡ„ tokenпјҢиҝҷдёӘеҖјжҜҸдёӘйӣҶзҫӨдҝқжҢҒе”ҜдёҖ
  иҝҷж ·пјҢеҰӮжһңдҪ иҰҒйҮҚж–°еҲӣе»әйӣҶзҫӨ
    еҚідҪҝй…ҚзҪ®е’Ңд№ӢеүҚдёҖж ·пјҢд№ҹдјҡеҶҚж¬Ўз”ҹжҲҗж–°зҡ„йӣҶзҫӨе’ҢиҠӮзӮ№ uuid
    еҗҰеҲҷдјҡеҜјиҮҙеӨҡдёӘйӣҶзҫӨд№Ӣй—ҙзҡ„еҶІзӘҒпјҢйҖ жҲҗжңӘзҹҘзҡ„й”ҷиҜҜ

еӨҮжіЁ

вҖ“init ејҖеӨҙзҡ„й…ҚзҪ®йғҪжҳҜеңЁз¬¬дёҖж¬ЎеҗҜеҠЁ etcd йӣҶзҫӨзҡ„ж—¶еҖҷжүҚдјҡз”ЁеҲ°пјҢеҗҺз»ӯиҠӮзӮ№зҡ„йҮҚеҗҜдјҡиў«еҝҪз•ҘгҖӮеҰӮжһңжңҚеҠЎе·Із»ҸиҝҗиЎҢиҝҮе°ұиҰҒжҠҠдҝ®ж”№ вҖ“initial-cluster-state дёә existing

вҖ“advertise-client-urls:

default: вҖңhttp://localhost:2379вҖқ

иҜҙжҳҺ:
  еҜ№еӨ–е…¬е‘Ҡзҡ„иҜҘиҠӮзӮ№е®ўжҲ·з«Ҝзӣ‘еҗ¬ең°еқҖпјҢиҝҷдёӘеҖјдјҡе‘ҠиҜүйӣҶзҫӨдёӯе…¶д»–иҠӮзӮ№
  List of this memberвҖҷs client URLs to advertise to the rest of the cluster

default: ""

Comma-separated list of supported TLS cipher suites between server/client and peers.
